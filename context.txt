# Cetus — HTML-to-Video Rendering Reference

Cetus renders HTML/CSS/JS compositions to video using headless Chrome + ffmpeg.

Docs: https://cetus.cenvero.org/docs/
If you hit the same validation error more than 3 times, read the docs page — it has the full reference.

---

## Correct HTML Composition Structure (READ THIS FIRST)

The composition root is a <div> INSIDE <body> with data-composition-id.
Do NOT put data attributes on <body>. Do NOT use <meta name="cetus:*"> tags — they are not parsed.

### Required attributes on the root <div>:
  data-composition-id   non-empty string identifier
  data-width            positive integer (pixels)
  data-height           positive integer (pixels)
  data-duration         positive float (seconds)

### Optional attributes on the root <div>:
  data-fps              positive integer (default: 30)

### Required attributes on every .clip element:
  data-start            non-negative float (seconds, when clip becomes visible)
  data-duration         positive float (seconds, how long clip is visible)
  data-track-index      integer (track layer, 0-based, controls z-order)

### Optional attributes on every .clip element:
  id                    string (recommended for GSAP targeting)
  data-volume           float 0.0–1.0 (default: 1.0)

### Constraint:
  data-start + data-duration must be <= composition data-duration
  (a clip cannot extend past the end of the composition)

## Complete Working Template

<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js"></script>
  <style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { margin: 0; }
    #root {
      width: 1920px;
      height: 1080px;
      overflow: hidden;
      background: #000;
      position: relative;
    }
    .clip { position: absolute; display: none; }
  </style>
</head>
<body>

  <div id="root"
       data-composition-id="my-comp"
       data-width="1920"
       data-height="1080"
       data-duration="10"
       data-fps="30">

    <div class="clip" id="title"
         data-start="1"
         data-duration="5"
         data-track-index="0"
         style="color:white; font-size:80px; top:400px; left:200px;">
      Hello World
    </div>

  </div>

  <script>
    const FPS = 30;
    const DURATION = 10; // must match data-duration above

    window.__timelines = [];

    // paused: true is REQUIRED — Cetus seeks timelines, never plays them
    const tl = gsap.timeline({ paused: true });

    // Show clip from 1s to 6s
    tl.set('#title', { display: 'block' }, 1)
    tl.from('#title', { opacity: 0, duration: 0.5 }, 1)
    tl.to('#title', { opacity: 0, duration: 0.5 }, 5.5)
    tl.set('#title', { display: 'none' }, 6)

    // CRITICAL: pin timeline to full duration or elements freeze at final state
    tl.set({}, {}, DURATION);

    // REQUIRED: register every timeline
    window.__timelines.push(tl);
  </script>
</body>
</html>

## The Seek Engine

Cetus does NOT record live playback. It seeks frame-by-frame. Every animation must be seekable.

Seek order for each frame:
1. Calls window.__cetusRenderFrame(frameIndex, fps) if defined
2. Calls tl.seek(cetusTime, false) on every timeline in window.__timelines[]
3. Sets .clip elements outside their time window to display: none
4. Calls functions in window.__cetusFrameHooks[]
5. Waits for pending fetch / Promise calls to settle
6. Waits for fonts and images to load
7. Takes the screenshot

Rule: Every animated element must be driven by a GSAP timeline registered in window.__timelines.
If it is not there, Cetus never seeks it — it is frozen at its CSS initial state in every frame.

---

## GSAP Timeline Rules (CRITICAL)

  // 1. Always use absolute time, never relative offsets
  tl.to(el, { opacity: 1, duration: 0.5 }, 1.0)   // CORRECT
  tl.to(el, { opacity: 1, duration: 0.5 }, "+=1")  // WRONG — breaks seek

  // 2. Register every timeline — REQUIRED
  window.__timelines = window.__timelines || [];
  window.__timelines.push(tl);

  // 3. CRITICAL: pin the timeline to the full composition duration
  // Without this, GSAP ends the timeline early and ALL elements freeze at their final state
  const DURATION = 10; // must equal data-duration on root div
  tl.set({}, {}, DURATION);

  // 4. Always create timelines paused — Cetus seeks them, never plays them
  // validator will ERROR if paused: true is missing
  const tl = gsap.timeline({ paused: true });

  // 5. Never use CSS transitions or @keyframes on seekable elements — use GSAP only

  // 6. Default clips to display:none in CSS (already handled if you use .clip class)

## Validation Rules (cetus validate)

cetus validate will ERROR if:
  - No element has data-composition-id
  - data-width, data-height, or data-duration missing from root
  - Any .clip is missing data-start, data-duration, or data-track-index
  - Any clip's start + duration exceeds composition duration
  - GSAP timeline created without paused: true
  - GSAP timeline created without window.__timelines.push(tl)
  - Any referenced local asset file does not exist

cetus validate will WARN if:
  - Remote URLs used (fonts.googleapis.com, cdnjs, etc.) — prefer local assets
  - No .clip elements found
  - Duplicate clip IDs
  - Elements positioned outside the frame bounds

Warnings do not block rendering. Errors do.

## Quality Guide

  CRF 0       Codec default (~CRF 23 for H.264). Good for most web and social videos.
  CRF 16      Mastering / highest quality. Large file.
  CRF 18      High quality delivery. Visually near-lossless. Recommended for archival.
  CRF 23      Balanced. Good starting point.
  CRF 28      Smaller file. Use for preview or upload-optimized delivery.
  CRF 35+     Heavy compression. Draft or thumbnail previews only.

4K output: --scale 4k --quality 16
WebM/VP9:  --format webm (VP9 default balanced CRF is ~30)

## Resume Workflow

Step 1 — Render with frame cache:
  cetus render cetus.html -o output.mp4 --frames-dir .cetus-frames --concurrency 2

Step 2 — If interrupted, resume:
  cetus render cetus.html -o output.mp4 --frames-dir .cetus-frames --resume --concurrency 2
  Already-captured frames are skipped instantly. Only missing frames are re-rendered.

Step 3 — Encode only when all frames are captured:
  cetus encode .cetus-frames -o output.mp4

When to use --frames-dir:
  < 300 frames   direct render, no --frames-dir needed
  300–599        use --frames-dir, --concurrency 1 or 2
  600+           use --frames-dir --concurrency 2 (strongly recommended)

--concurrency requires --frames-dir. Each worker opens a separate Chrome instance.

## Common Mistakes

1. Putting data attributes on <body> instead of a <div> inside body
2. Using <meta name="cetus:fps"> — these are NOT parsed by Cetus
3. Missing data-track-index on .clip elements — it is REQUIRED
4. Relative GSAP positions (+=1, <, -=0.5) — use absolute seconds always
5. Missing tl.set({},{},DURATION) — timeline ends early, elements freeze at final state
6. Not pushing to window.__timelines — timeline is never seeked
7. No paused: true on timeline — validator errors, seek artifacts
8. CSS transition or @keyframes on seekable elements — does not respond to JS seek
9. clip start + duration exceeds composition duration — validation error
10. Using cetus preview to check specific frames — preview only shows t=0; use cetus seek

## Debugging Checklist

- cetus validate cetus.html — always run before rendering
- If same error appears 3+ times: read https://cetus.cenvero.org/docs/
- cetus seek cetus.html --at 2s -o check.png — inspect any frame without full render
- Black screen → no timelines registered, or all elements display:none
- Frozen frame → missing tl.set({},{},DURATION)
- Element invisible → missing tl.set({display:'block'}, startTime) or wrong data-start
- Wrong timing → check all positions are absolute seconds
- Validation error on clip → check data-start, data-duration, data-track-index all present
- Render stopped → use --resume to continue

## CLI Commands

cetus completion — Generate the autocompletion script for the specified shell

cetus completion bash — Generate the autocompletion script for bash
  --no-descriptions	disable completion descriptions


cetus completion fish — Generate the autocompletion script for fish
  --no-descriptions	disable completion descriptions


cetus completion powershell — Generate the autocompletion script for powershell
  --no-descriptions	disable completion descriptions


cetus completion zsh — Generate the autocompletion script for zsh
  --no-descriptions	disable completion descriptions


cetus context — Print the full Cetus AI context (seek engine docs, GSAP rules, CLI reference)
  --help	help for context

cetus encode — Encode a frame cache directory into a video (or extract a thumbnail)
  --audio STRING	audio file to mux into the output (mp3, wav, m4a, aac, flac, ogg, opus)
  --audio-fade-in FLOAT64	audio fade-in duration in seconds
  --audio-fade-out FLOAT64	audio fade-out duration in seconds
  --audio-loop	loop audio until the video duration is reached
  --audio-start FLOAT64	start offset into the audio file in seconds
  --audio-volume FLOAT64	audio volume from 0.0 to 1.0 (default: 1)
  --format STRING	output format: mp4 or webm (defaults from file extension)
  --fps INT	override frame rate from config.cetus
  --keep-frames	keep the frame cache directory after encoding
  --output STRINGARRAY	output file path; can be specified multiple times for multiple outputs
  --progress-format STRING	progress output format: text or json (default: text)
  --quality INT	encoder CRF quality (0 = codec default)
  --scale STRING	scale output resolution: 480p, 720p, 1080p, 4k, or WxH
  --subtitles STRING	subtitle file to burn into the video (SRT or ASS)
  --thumbnail STRING	extract a single frame at this timestamp (e.g. 5s, 1:30, 01:02:30)
  --timeout INT	max encode time in seconds; 0 disables deadline

cetus help — Help about any command

cetus preview — Serve an HTML composition locally with live reload
  --no-open	do not open a browser automatically
  --port INT	port to listen on

cetus render — Render an HTML composition to MP4 or WebM
  --audio STRING	audio file to mux into the output
  --audio-fade-in FLOAT64	audio fade-in duration in seconds
  --audio-fade-out FLOAT64	audio fade-out duration in seconds
  --audio-loop	loop audio until the render duration is reached
  --audio-start FLOAT64	audio start time in seconds on the render timeline
  --audio-volume FLOAT64	audio volume from 0.0 to 1.0 (default: 1)
  --concurrency INT	parallel frame capture workers when --frames-dir or --resume is used (default: 1)
  --format STRING	output format: mp4 or webm
  --fps INT	frames per second (default: 30)
  --frames-dir STRING	directory for cached PNG frames
  --height INT	output height in pixels
  --keep-frames	keep the frame cache directory after a successful render
  --no-gpu	disable GPU acceleration
  --output STRING	output file path
  --progress-format STRING	progress output format: text or json (default: text)
  --quality INT	encoder CRF quality (0 = codec default; lower = better quality, larger file)
  --resume	reuse existing frames from --frames-dir; defaults to .cetus-frames when no directory is set
  --scale STRING	scale output resolution: 480p, 720p, 1080p, 4k, or WxH (e.g. 1920x1080)
  --subtitles STRING	subtitle file to burn into the video (SRT or ASS)
  --timeout INT	max render time in seconds; 0 disables the total render deadline
  --width INT	output width in pixels

cetus seek — Render a single frame from a composition to a PNG
  --at STRING	timestamp to capture (e.g. 5s, 1:30, 01:02:30) (default: 0s)
  --fps INT	override composition FPS
  --height INT	override composition height
  --no-gpu	disable GPU acceleration
  --output STRING	output PNG file path
  --timeout INT	max time in seconds; 0 disables the deadline
  --width INT	override composition width

cetus update — Check for and apply Cetus updates

cetus update apply — Download and install the latest Cetus release
  --force	apply the latest release even if the current version is already current


cetus update check — Check whether a Cetus update is available


cetus validate — Validate a Cetus HTML composition before rendering

cetus version — Print version information

cetus watch — Re-render a composition whenever the source files change
  --audio STRING	audio file to mux into the output
  --audio-fade-in FLOAT64	audio fade-in duration in seconds
  --audio-fade-out FLOAT64	audio fade-out duration in seconds
  --audio-loop	loop audio until the render duration is reached
  --audio-start FLOAT64	audio start offset in seconds
  --audio-volume FLOAT64	audio volume from 0.0 to 1.0 (default: 1)
  --concurrency INT	parallel frame capture workers (requires --frames-dir) (default: 1)
  --format STRING	output format: mp4 or webm
  --fps INT	override composition FPS
  --frames-dir STRING	directory for PNG frames; speeds up re-renders by reusing the Chrome render path
  --height INT	override composition height
  --no-gpu	disable GPU acceleration
  --output STRING	output file path (overwritten on each re-render)
  --progress-format STRING	progress output format: text or json (default: text)
  --quality INT	encoder CRF quality (0 = codec default)
  --scale STRING	scale output: 480p, 720p, 1080p, 4k, or WxH
  --subtitles STRING	subtitle file to burn into the video (SRT or ASS)
  --timeout INT	per-render time limit in seconds; 0 disables the deadline
  --width INT	override composition width
